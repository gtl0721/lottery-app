# Timmy & Emily 婚禮大樂透使用說明

本專案為單機版婚禮樂透工具，不需要安裝後端、不需要網路，也不需要部署網站。

專案包含兩個頁面：

| 用途 | 頁面 |
| --- | --- |
| 婚宴現場開獎 | `index.html` |
| 列印賓客兌獎卡 | `cards.html` |

## 1. 使用前準備

建議使用 Chrome 或 Edge 開啟頁面。

### 直接開啟

在檔案總管進入 `D:\lottery-app`，雙擊對應的 HTML 檔案。

也可以在瀏覽器網址列輸入：

```text
file:///D:/lottery-app/index.html
```

### 使用 VS Code Live Server

1. 使用 VS Code 開啟 `D:\lottery-app`。
2. 安裝 `Live Server` 擴充套件。
3. 在 `index.html` 上按右鍵。
4. 選擇 `Open with Live Server`。
5. 瀏覽器會開啟類似以下網址：

```text
http://127.0.0.1:5500/index.html
```

兌獎卡頁面網址為：

```text
http://127.0.0.1:5500/cards.html
```

## 2. 婚宴現場開獎

開啟 `index.html` 後，建議按 `F11` 進入瀏覽器全螢幕模式，讓畫面適合投影。

### 隨機開獎

1. 確認管理模式保持隱藏。
2. 點擊 `開始開獎`。
3. 每次點擊只會揭露 1 顆號碼球。
4. 繼續點擊按鈕，直到 6 顆號碼全部揭露。
5. 完成後畫面會顯示 `開獎完成，中獎請舉手!!!`。

系統會從 `1` 到 `36` 中隨機抽出 6 個不重複號碼，最後依照小到大顯示。

### 指定號碼開獎

管理模式預設隱藏，避免賓客看到設定內容。

1. 按鍵盤 `M` 顯示管理模式。
2. 在 `指定號碼1` 輸入第一組 6 個號碼。
3. 在 `指定號碼2` 輸入第二組 6 個號碼。
4. 再按一次 `M` 隱藏管理模式。
5. 點擊 `開始開獎`。
6. 第一組開完後，點擊 `繼續開獎` 開始第二組。

輸入範例：

```text
指定號碼1：3,8,12,19,25,36
指定號碼2：1,6,14,20,28,33
```

指定號碼規則：

- 每組必須剛好輸入 6 個號碼。
- 號碼範圍必須為 `1` 到 `36`。
- 同一組號碼不可重複。
- 使用指定號碼時，兩組都必須填寫。
- 系統會先開 `指定號碼1`，再開 `指定號碼2`。

若兩組指定號碼皆為空白，系統會改用隨機開獎。

## 3. 列印婚禮大樂透兌獎卡

開啟 `cards.html` 後，頁面會載入全部賓客兌獎卡。

每張卡片包含：

- `婚禮大樂透`
- `Timmy & Emily Wedding Lottery`
- 6 個賓客對獎號碼
- 開獎時間 `2026-06-14`

版面設定：

- 每張卡尺寸為 A7。
- 每張 A4 紙張排版 8 張 A7 卡。
- 卡片使用紅金婚禮風格。

### 建議列印設定

1. 點擊頁面右上角的 `列印兌獎卡`，或按 `Ctrl + P`。
2. 紙張尺寸選擇 `A4`。
3. 方向選擇 `直向`。
4. 邊界選擇 `無`。
5. 縮放比例選擇 `100%` 或 `預設`。
6. 開啟 `背景圖形`，才能列印紅金色背景。
7. 先列印或另存前 1 頁 PDF，確認裁切位置後再印全部卡片。

## 4. 兌獎卡資料來源

賓客兌獎號碼來源檔案：

```text
assets/wedding_lottery_guest_numbers_no_cross_winners.xlsx
```

使用資料位置：

```text
分頁：賓客對獎號碼
欄位：C 欄 賓客對獎號碼
```

瀏覽器實際載入的資料存放在：

```text
cards-data.js
```

若 Excel 內容有更新，需要同步重新產生 `cards-data.js` 後再列印。

## 5. 常見狀況

### 管理模式沒有出現

請先點一下開獎頁空白處，再按鍵盤 `M`。

### 無法開始指定號碼開獎

請檢查兩組指定號碼是否都已填寫，且每組都有 6 個不重複的 `1` 到 `36` 整數。

### 列印卡片沒有紅金背景

請在瀏覽器列印設定中開啟 `背景圖形`。

### 投影畫面未滿版

請按 `F11` 進入全螢幕模式。頁面會依照螢幕尺寸等比例縮放。

### 現場操作建議

婚宴開始前先測試一次隨機開獎與指定號碼開獎，並確認投影畫面、鍵盤 `M` 快捷鍵和滑鼠點擊都能正常使用。
